# 小红书章节描述升级设计

## 背景

小红书章节弹窗新增了每章“章节描述”输入框，限制为 100 字。弹窗顶部同时新增“内容总结”，它属于整段视频，不属于单条章节。

## 设计

- 小红书章节名继续限制为 11 字。
- 通用章节数据中的 `summary` 写入每条章节的“章节描述”，限制为 100 字。
- 不自动填写弹窗顶部的“内容总结”。
- 新版页面在保存前按时间、标题、描述三项复核，未保留的描述会进入现有重试流程。
- 旧版页面如果不存在“章节描述”输入框，仍只按时间和标题复核，保持兼容。
- 错误日志增加预期描述和页面实际描述，便于定位写入丢失问题。

## 验证

- 平台配置测试覆盖小红书 11 字标题和 100 字描述限制。
- 数据策略测试覆盖描述保留、超长报错和自动截断。
- DOM 策略测试覆盖描述不一致检测，以及旧页面忽略描述的兼容行为。
- 完整执行测试、语法检查和扩展打包。
