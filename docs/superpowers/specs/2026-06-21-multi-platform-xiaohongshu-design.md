# 多平台与小红书章节支持设计

## 背景

当前插件已支持抖音创作者中心的视频章节批量写入，但项目命名、manifest、README、Chrome 商店文案和 content script 都偏向“抖音专用”。用户希望首个公开发布版本同时支持抖音和小红书，因此需要先升级为多平台架构，再发布到 GitHub 和 Chrome 应用商店。

2026-06-21 对小红书创作服务平台页面做过只读检查，目标页面是：

`https://creator.xiaohongshu.com/publish/publish?from=menu&target=video`

页面已上传视频时，章节入口位于“内容设置 / 添加章节 / 去添加”。点击后出现独立章节弹窗。弹窗结构：

- 左侧是视频预览和时间轴。
- 右侧标题为“章节信息”。
- 底部有“+ 添加章节”“取消”“保存章节”按钮。
- 点击“+ 添加章节”后，会新增一条章节编辑行。
- 单条编辑行包含：
  - 分钟输入框：`input.time-field.minutes-field`，`maxlength=4`。
  - 秒输入框：`input.time-field`，`maxlength=2`。
  - 章节名输入框：`input.title-field[placeholder="输入章节名称"]`，`maxlength=14`。
- 小红书章节没有简介字段。
- 页面存在可读时长的 `video` 元素，当前样例视频时长约 288 秒。

## 目标

- 插件从“抖音视频章节助手”升级为“创作者章节助手”。
- 首发公开版本支持抖音和小红书。
- 保留现有抖音能力。
- 新增小红书章节写入能力。
- popup 根据当前标签页自动识别平台，并展示平台限制。
- 解析器仍支持同一份章节文本格式。
- 平台 adapter 负责把通用章节数据写入不同创作者中心页面。

## 非目标

- 不通过小红书 API 发布或编辑内容。
- 不点击小红书最终发布按钮。
- 不保存小红书章节弹窗之外的发布设置。
- 不支持除抖音、小红书以外的平台。
- 不把小红书简介写入其他字段；第一版只使用章节名。

## 命名

插件显示名改为：**创作者章节助手**

原因：

- 不绑定抖音或小红书。
- 适合后续扩展到 B 站、视频号等平台。
- 比“视频章节助手”更明确面向创作者中心工作流。

包名和 GitHub 仓库名建议后续改为：

- `creator-chapter-helper`

## 架构

新增平台适配层：

- `src/platforms/platform-registry.js`
  - 判断当前页面属于哪个平台。
  - 暴露平台配置：平台名、host、章节标题限制、是否支持简介。
- `src/content/platform-progress.js`
  - 复用页面右下角进度面板。
- `src/content/douyin-adapter.js`
  - 从现有 `douyin.js` 中拆出抖音页面自动化。
- `src/content/xiaohongshu-adapter.js`
  - 实现小红书页面自动化。
- `src/content/index.js`
  - 统一接收 popup 消息。
  - 按平台分发到 adapter。

保留通用模块：

- `src/lib/chapter-parser.js`
- `src/lib/text-policy.js`
- `src/lib/extension-runtime.js`

## 小红书数据策略

解析输入：

`- 00:05-00:28｜阶段二总结：把规划基石阶段串成一张考试作战地图。`

对小红书写入：

- 时间点：使用开始时间 `00:05`。
- 章节名：优先使用解析出的 `title`。
- 简介：不写入。
- 标题长度限制：11 字。

如果标题为空：

- 使用简介前 11 字作为章节名，并提示警告。

如果标题超过 11 字：

- 未启用截断时阻止写入。
- 启用截断时截断到 11 字。

## 小红书自动化流程

1. 确认当前页面是小红书视频发布页。
2. 找到“添加章节”区域里的“去添加”入口。
3. 点击“去添加”，打开章节弹窗。
4. 读取视频时长，校验章节开始时间是否在视频时长内。
5. 如果选择“清空已有章节”：
   - 通过章节列表中的删除控件或重建列表方式清空已有章节。
   - 如果页面没有暴露可稳定删除控件，则第一版提示用户手动清空后重试。
6. 对每条章节：
   - 点击“+ 添加章节”。
   - 找到最新一条章节编辑行。
   - 填写分钟、秒和章节名。
   - 校验行内值已写入。
7. 点击“保存章节”。
8. 校验弹窗关闭或主页面章节状态更新。
9. 不触碰最终发布按钮。

## 风险

- 小红书章节已有条目的删除控件在初次探测中不明显，清空已有章节可能需要后续针对已有章节状态再探测。
- 小红书章节名限制 11 字，与抖音 12 字不同，必须按平台校验。
- 小红书时间输入是分钟/秒分离字段，不是统一时间戳输入。
- 小红书弹窗内视频可能会自动播放，脚本应尽量不依赖播放状态。

## 测试策略

单元测试：

- 平台识别：抖音、小红书、未知页面。
- 平台限制：抖音 12 字，小红书 11 字。
- 小红书章节投影：从通用章节生成 `{ minutes, seconds, title }`。
- 小红书标题为空时从简介生成标题。
- 小红书标题超长时按选项报错或截断。

人工验证：

- 抖音现有导入流程仍可用。
- 小红书页面打开“去添加”后可批量新增章节。
- 小红书弹窗保存后，主页面章节区域显示已添加状态。
- 两个平台都不会点击最终发布按钮。
