# 抖音视频章节助手 Chrome 插件设计

## 背景

当前项目目录是一个新的非 git 工作区。第一版只支持抖音创作者中心的视频发布页：

`https://creator.douyin.com/creator-micro/content/post/video?enter_from=publish_page`

2026-06-21 对你当前打开的页面做过只读检查，确认章节编辑入口位于“视频章节 / 去编辑”。章节编辑弹窗具备以下关键结构：

- “输入章节名”是原生 `input`，限制 12 个字。
- “输入章节简介”是原生 `textarea`，限制 100 个字。
- 弹窗内存在可读写 `currentTime` 的 HTML `video` 元素。
- 单条章节的“确定”是自定义 `div` 控件，不是原生按钮。
- 弹窗底部“保存”是原生按钮，在章节发生变化后才会启用。

抖音开放平台有上传视频和创建视频作品相关 API，但公开的创建视频参数里暂未发现视频章节字段。API 方案还需要能力申请、平台审核和 OAuth 授权。因此第一版应自动化已有创作者中心页面，而不是替代整个发布流程。

## 目标

- 用户粘贴章节文本后，可以一键写入抖音创作者中心章节编辑器。
- 支持当前常用格式，例如：

  `- 00:05-00:28｜阶段二总结：把规划基石阶段串成一张考试作战地图。`

- 使用时间范围的开始时间作为抖音章节时间点。
- 写入前展示解析预览。
- 写入前校验抖音限制：
  - 章节名最多 12 个字。
  - 章节简介最多 100 个字。
  - 时间点必须落在当前视频时长内。
- 自动保存章节弹窗里的章节改动。
- 绝不点击最终“发布”按钮。

## 非目标

- 第一版不通过抖音开放平台 API 发布视频。
- 不处理上传视频、封面、合集、可见范围、定时发布等发布配置。
- 第一版不支持小红书或其他平台。
- 不使用后端服务。
- 不绕过抖音页面限制、账号权限或平台校验。

## 用户体验

插件在匹配的抖音创作者中心页面上提供浏览器按钮弹窗。

弹窗包含：

- 用于粘贴章节原文的大文本框。
- 解析后的预览表，展示开始时间、结束时间、章节名、章节简介。
- 校验提示，包括时间格式错误、重复时间点、时间倒序、超出平台字数限制等。
- 选项：
  - 写入前清空已有章节。
  - 自动截断章节名到 12 个字。
  - 自动截断章节简介到 100 个字。
  - 仅预览，不写入页面。
- 主操作按钮：“写入抖音章节”。

执行过程中，content script 会在抖音页面内注入一个轻量进度面板。这样即使插件弹窗关闭，用户也能看到当前步骤、正在写入哪条章节、完成数量和错误信息。

## 架构

插件使用 Chrome Manifest V3，不需要后端服务。

主要文件职责：

- `manifest.json`
  - 声明权限。
  - 匹配 `https://creator.douyin.com/*`。
- `src/popup/*`
  - 渲染粘贴区、预览表和选项。
  - 将校验后的章节数据发送给当前标签页。
- `src/content/douyin.js`
  - 运行在创作者中心页面。
  - 检测章节入口和章节弹窗。
  - 执行页面自动化写入。
  - 注入并更新页面进度面板。
- `src/lib/chapter-parser.js`
  - 将粘贴文本解析成结构化章节。
  - 规范化时间格式和分隔符。
  - 输出警告和错误。
- `src/lib/text-policy.js`
  - 处理章节名和章节简介长度规则。
  - 保持截断行为明确、可测试。

解析器和文本策略不依赖 Chrome API，便于用 Node 单元测试覆盖。

## 数据模型

```ts
type ParsedChapter = {
  rawLine: string;
  lineNumber: number;
  startSeconds: number;
  endSeconds?: number;
  title: string;
  summary: string;
  warnings: string[];
  errors: string[];
};

type ImportOptions = {
  replaceExisting: boolean;
  truncateTitle: boolean;
  truncateSummary: boolean;
  dryRun: boolean;
};
```

## 解析规则

解析器按“每行一条章节”处理。

支持的时间格式：

- `00:05-00:28`
- `00:05 - 00:28`
- `1:05-1:28`
- `01:02:03-01:03:20`

支持时间范围后的分隔符：

- `|`
- `｜`
- 空白字符

时间范围后的内容按第一个 `:` 或 `：` 切分章节名和简介。

如果某一行没有章节名和简介分隔符：

- 章节名默认取剩余文本的前 12 个字。
- 简介默认使用剩余全文。
- 预览里展示警告。

解析器会忽略时间戳前的项目符号，例如 `-`、`*`、`1.`、中文序号符号等。

## 抖音页面自动化流程

1. 确认当前标签页是支持的抖音创作者中心视频发布页。
2. 找到“视频章节”区域并点击“去编辑”。
3. 等待章节弹窗出现。
4. 必要时切换到“手动添加”模式。
5. 从弹窗内的 `video` 元素读取视频时长。
6. 校验所有章节时间点是否落在视频时长内。
7. 如果用户启用“写入前清空已有章节”：
   - 在已有章节存在时点击“清空”。
   - 如果抖音出现本地确认框，确认清空。
8. 逐条写入章节：
   - 将弹窗内 `video.currentTime` 设置为章节开始秒数。
   - 派发媒体和输入相关事件，让抖音页面感知 seek。
   - 填写“输入章节名”。
   - 填写“输入章节简介”。
   - 点击单条章节的自定义“确定”控件。
   - 校验右侧章节列表出现新增章节。
9. 点击弹窗底部“保存”。
10. 校验主页面章节摘要更新。
11. 不触碰最终“发布”按钮。

## 错误处理

只要存在解析或校验错误，插件应在改动页面前停止。

页面自动化阶段的可恢复错误应显示在进度面板中，并给出清晰动作建议：

- 找不到章节入口：提示用户确认视频已上传，且页面已显示视频章节功能。
- 找不到章节弹窗：提示用户手动点击“去编辑”后重试。
- 找不到视频元素：提示用户重新打开章节编辑弹窗。
- 时间点超出视频时长：显示出错行号。
- 章节名或简介过长：提示用户启用自动截断或手动修改。
- 保存按钮一直未启用：提示页面没有接受改动，并保留弹窗供用户检查。

自动化失败时不主动关闭弹窗，方便用户查看现场状态。

## 测试策略

单元测试：

- 解析器能处理用户样例格式。
- 解析器能处理 `|`、`｜`、空格、项目符号和 `hh:mm:ss`。
- 解析器能报告错误时间格式。
- 文本策略能校验 12 字和 100 字限制。
- 自动截断选项行为明确且稳定。

集成和人工验证：

- 在 Chrome 中加载未打包插件。
- 打开已上传视频的抖音创作者中心发布页。
- 粘贴样例章节列表。
- 执行仅预览。
- 启用“清空已有章节”后执行写入。
- 确认弹窗章节数量与解析章节数量一致。
- 确认最终“发布”按钮没有被点击。

## 风险

- 抖音可能修改类名或弹窗结构。实现应优先使用可见文本、placeholder 和弹窗作用域查询，再回退到类名。
- 单条章节“确定”不是原生按钮，需要稳健的点击辅助函数和点击后校验。
- 直接设置 `video.currentTime` 未必总能更新抖音内部状态。如果不可靠，需要回退到按比例点击进度条坐标。
- 插件弹窗可能在执行中关闭，所以需要页面内进度面板承接执行状态。
- 已有章节可能包含用户手动编辑内容，因此“清空已有章节”必须是显式选项。

## 实施顺序

1. 建立解析器和文本策略模块，并先写单元测试。
2. 建立弹窗 UI 和预览状态。
3. 建立抖音页面检测和章节弹窗打开逻辑。
4. 实现真实抖音弹窗的手动章节写入流程。
5. 增加页面进度面板和错误报告。
6. 用用户当前样例验证。
7. 打包为可在 Chrome 本地加载的未打包插件。
